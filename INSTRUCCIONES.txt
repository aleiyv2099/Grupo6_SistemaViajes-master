  SISTEMA DE AGENCIA DE VIAJES - GRUPO 6
  Instrucciones para ejecutar el proyecto (desde el ZIP)
=====================================================

PROGRAMAS QUE NECESITAS TENER INSTALADOS
-----------------------------------------
1. Java 17 o superior
   https://adoptium.net/

2. MySQL Server 8 (solo el servidor)
   https://dev.mysql.com/downloads/mysql/

3. NetBeans IDE (para abrir y ejecutar con un boton)
   https://netbeans.apache.org/


IMPORTANTE - LO UNICO QUE HAY QUE PREPARAR ES LA BASE DE DATOS
---------------------------------------------------------------
La aplicacion crea sus TABLAS sola la primera vez que arranca.
PERO se conecta con un usuario de MySQL llamado "viajes", y ese
usuario NO se crea solo: hay que crearlo UNA vez importando el
archivo SQL que viene en el ZIP. Eso es todo.


PASOS
------

PASO 1 - Extrae el ZIP en cualquier carpeta.

PASO 2 - Asegurate de que MySQL este ENCENDIDO.
   Linux:  sudo systemctl start mysql
   Windows/Mac: inicia el servicio MySQL o abre MySQL Workbench.

PASO 3 - Crea la base de datos y el usuario (el paso clave).
   Elige UNA opcion:

   -- OPCION A: Linux, con el script automatico --
       bash setup.sh                    (si tu root NO tiene contrasena)
       bash setup.sh -p TU_PASS_ROOT    (si tu root tiene contrasena)
       sudo bash setup.sh               (si tu root necesita sudo)

   -- OPCION B: WINDOWS con "MySQL Command Line Client" (lo mas simple) --
   1. Menu Inicio -> abre "MySQL 8.0 Command Line Client".
   2. Escribe la contrasena de root de MySQL.
   3. En el prompt  mysql>  escribe (con la ruta real donde extrajiste
      el ZIP y usando barras normales "/"):

       source C:/Users/TuUsuario/Downloads/Grupo6_SistemaViajes-master/SistemaReserva.sql

   -- OPCION C: WINDOWS con MySQL Workbench (grafica) --
   1. Abre MySQL Workbench y conectate a tu servidor local (como root).
   2. Menu: File -> Open SQL Script -> selecciona  SistemaReserva.sql
   3. Pulsa el rayo (Execute) para ejecutar todo el script.
   (No uses "Data Import"; "Open SQL Script + Execute" es mas confiable.)

   -- OPCION D: Linea de comandos (Linux, Mac o CMD de Windows) --
       mysql -u root -p < SistemaReserva.sql
   OJO: en PowerShell de Windows el simbolo "<" NO funciona; usa la
   Opcion B o C, o el CMD clasico.

   Cualquiera de las opciones crea:
     - La base de datos "sistemareserva" con todas las tablas y datos de ejemplo.
     - El usuario "viajes" con contrasena "Viajes123#".

PASO 4 - NO tienes que tocar la configuracion.
   El archivo src/main/resources/database.properties YA viene configurado
   para el usuario "viajes". No cambies nada.

PASO 5 - Ejecuta la aplicacion.

   -- Con NetBeans --
   Archivo -> Abrir proyecto -> selecciona la carpeta del proyecto
   Espera que descargue las dependencias (solo la primera vez, requiere internet)
   Presiona Run (triangulo verde) o F6.

   -- Por terminal (genera un .jar ejecutable) --
       mvn clean package -DskipTests
       java -jar target/SistemaAgenciaViajesGrupo6-1.0-SNAPSHOT.jar

   (Se usa -DskipTests porque el proyecto trae unos tests de integracion
    que dependen de datos concretos; no afectan a la app.)

SI ALGO NO FUNCIONA
--------------------
- "Access denied for user 'viajes'"  -> no hiciste el PASO 3. Importa el SQL.
- "Table '...soporte' doesn't exist"  -> base de datos vieja: vuelve a importar el SQL del PASO 3.
- La app no abre / error de conexion  -> MySQL no esta encendido.
- "mvn package" falla en los tests    -> usa  mvn clean package -DskipTests
